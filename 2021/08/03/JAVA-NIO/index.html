JAVA-NIO

代码仓库

IO和NIO区别：

第一点：

  • IO面向流：单向
  • NIO面向缓冲区：双向

第二点：

  • IO是阻塞IO
  • NIO是非阻塞IO

第三点：

  • NIO有选择器

NIO的通道负责传输，缓冲区负责存储

缓冲区

一：缓冲区就是数组，用于存储不同类型的数据。

根据类型不同（boolean除外），提供相应类型的缓冲区

  • BytebBuffer
  • CharBuffer
  • ShortBuffer
  • IntBuffer
  • LongBuffer
  • FloatBuffer
  • DoubleBuffer

一般都通过allocate()获取缓冲区

二：缓冲区存取数据的两个核心方法：

  • push:存入数据到缓冲区

  • get:获取的缓冲区的数据

三：缓冲区的四个核心属性：

  • capactity:容量，表示缓冲区中最大存储数据的容量，一旦声明就不能改变。
  • limit:界限，表示缓冲区可以操作数据的大小。（limit后数据不能进行读写）
  • position:位置，表示缓冲区中正在操作数据的位置。
  • mark:标记，表示记录当前postion位置，可以通过reset()恢复到mark位置

直接缓冲区和非直接缓冲区

非直接缓冲区：通过allocate()分配的缓冲区，将建立在JVM的内存中

直接缓冲区：通过allocateDirect()分配直接缓冲区，将建立在物理内存中，可以提高效率，但是消耗资源，有时候不稳定

通道

用于源节点与目标节点的连接

在Java NIO中负责缓冲区数据的传输

一：通道的主要实现类

java.nio.channels.Channel接口：

  • FileChannel 本地
  • SocketChannel TCP
  • ServerSocketChannel TCP
  • DatagramChannel UDP

二：获取通道

  1. Java针对支持通道的类提供了getChannel

  • 本地IO:

    • FileInputStream/FileOutStream

    • RandomAccessFile

  • 网络IO:

    • Socket
    • ServerSocket
    • DatagramSocket
  1. 在jdk7中的NIO.2针对各个通道提供了静态方法open()
  2. 在jdk7中的NIO.2的Files工具类的newByteChannel()

三：通道之间的数据传输

  • transferFrom
  • transferTo

分散与聚集

分散（Scatter）读取：将通道中的数据分散到多个缓冲区中

聚集（Gather）写入：将多个缓冲区的数据聚集到通道中

字符集Charset

编码：字符串 -> 字节数组

解码：字节数组 -> 字符串

网络通信

三个核心：

  • 通道(Channel)：负责连接

    • java.nio.channels.Channel接口：

      • SelectableChannel

        • SocketChannel
        • ServerSocketChannel
        • DatagramChannel
        • Pipe.SinkChannel
        • Pipe.SourceChannel

  • 缓冲区(Buffer)：负责数据的存取

  • 选择器(Selector)：是SelectableChannel的多路复用器。用于监控SelectableChannel的IO状况
